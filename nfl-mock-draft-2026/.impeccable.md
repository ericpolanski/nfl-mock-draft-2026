## Design Context

### Users
NFL fans and fantasy football enthusiasts who want to experience what it's like to run an NFL draft. Users range from casual viewers watching the draft on TV to serious analysts who enjoy deep player analysis. They typically use this on desktop or tablet while watching the actual NFL draft.

### Brand Personality
Sports entertainment meets modern web app. The interface should feel like a professional broadcast dashboard - exciting when picks happen but clean enough for extended use. Three words: **Professional**, **Energetic**, **Immersive**.

### Aesthetic Direction
- **Theme**: Dark mode primarily (sports broadcast aesthetic)
- **Base colors**: Slate grays (#1e293b, #0f172a) for backgrounds
- **Accent**: Cyan (#00fff5) for interactive elements and highlights
- **Typography**: Inter font family, clean sans-serif
- **Visual style**: Card-based components with subtle borders, gradient accents, smooth animations for pick updates

### Design Principles
1. **Team Colors First**: Each NFL team's primary color should be prominently displayed for their picks - this creates instant recognition and emotional connection
2. **Information Density**: Show enough data to satisfy analysts (prospect grades, team needs, college stats) without overwhelming casual users
3. **Real-time Feel**: Animations and transitions that make the draft feel alive - pulsing indicators, smooth scrolling to current pick
4. **Clear Hierarchy**: Current pick always most prominent, followed by available players, then historical picks
5. **Mobile Responsive**: Draft room works on tablet, though optimal experience is desktop

### Tech Stack Constraints
- React 18 + Vite
- Tailwind CSS (use slate-* color palette, avoid pure blacks/grays)
- No custom fonts needed (Inter is already loaded)
